### 2. 主要模块说明

| 模块 | 职责 |
|------|------|
| **Main** | 应用入口，初始化各模块 |
| **Lark** | 飞书 API 集成（消息接收/发送） |
| **Gateway** | 消息路由和工作流触发 |
| **Middleware** | 会话管理、去重、上下文构建 |
| **Queue** | 任务队列管理 |
| **Worker** | 任务执行引擎 |
| **Agent** | Claude CLI 代理 |
| **Skills** | 技能注册和加载 |
| **Session** | 会话和记忆管理 |
| **Stats** | 每日统计 |
| **DB** | SQLite 数据库 |
| **Reflection** | 反思引擎 |
| **Evolution** | 进化引擎 |
| **Workflow** | 工作流引擎 |

### 3. 数据流

1. 用户 → Lark (飞书)
2. Lark → Gateway → Middleware
3. Middleware → Queue
4. Queue → Worker
5. Worker → Agent/Skills/Session
6. 各模块 → DB

## PlantUML 文件结构

文件将保存为 `architecture.puml`，包含：

- 主要模块作为组件
- 模块间的依赖关系（使用箭头）
- 数据流方向
- 按功能分组的包

## 实现步骤

1. 创建 `architecture.puml` 文件
2. 定义主要模块和包
3. 添加模块间的关系
4. 验证图表正确性
