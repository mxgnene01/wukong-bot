# 代码行数统计工具设计文档

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 创建一个单文件脚本，统计 src 目录下 TypeScript 代码的有效行数，支持按目录细分统计。

**Architecture:** 单文件 Bun 脚本，包含文件扫描、分析、聚合和输出格式化功能。

**Tech Stack:** Bun, TypeScript, Node.js fs/path 模块

---

## 概述

本文档描述了 `count-lines.ts` 脚本的设计和实现方案，该脚本用于统计 src 目录下 TypeScript 代码的行数统计。

## 功能需求

### 核心功能
1. 递归扫描 `src/` 目录下所有 `.ts` 文件（排除 `.d.ts`）
2. 统计每个文件的：
   - 总行数
   - 有效代码行数（排除空行和单行注释）
   - 空行数
   - 注释行数
3. 按目录分组统计
4. 显示总体统计摘要

### 输出格式
- 总体统计在前
- 按目录排序的详细统计在后
- 美观的表格对齐格式

## 架构设计

### 单文件结构
所有功能在 `count-lines.ts` 一个文件中实现，包含以下核心组件：

```
count-lines.ts
├── 类型定义
├── 文件扫描器
├── 文件分析器
├── 目录聚合器
├── 输出格式化器
└── 主函数
```

## 数据流程

```
src/ 目录
    ↓
文件扫描器 (findAllTsFiles)
    ↓ (文件列表)
文件分析器 (analyzeFile)
    ↓ (文件统计数组)
目录聚合器 (按目录分组)
    ↓ (目录统计 + 总体统计)
输出格式化器
    ↓
控制台输出
```

## 详细设计

### 1. 类型定义

```typescript
interface FileStats {
  path: string;
  totalLines: number;
  codeLines: number;
  emptyLines: number;
  commentLines: number;
}

interface DirectoryStats {
  name: string;
  fileCount: number;
  totalLines: number;
  codeLines: number;
  emptyLines: number;
  commentLines: number;
}
```

### 2. 文件扫描器

**函数名**: `findAllTsFiles(dir: string, fileList: string[] = []): string[]`

**功能**: 递归扫描目录，收集所有 `.ts` 文件（排除 `.d.ts`）

**实现细节**:
- 使用 `readdirSync` 读取目录内容
- 使用 `statSync` 判断是文件还是目录
- 递归处理子目录
- 收集符合条件的文件路径

### 3. 文件分析器

**函数名**: `analyzeFile(filePath: string): FileStats`

**功能**: 逐行分析文件，统计各类行数

**行判断逻辑**:
- **空行**: `line.trim().length === 0`
- **注释行**: 以 `//` 开头，或以 `/*` 开头且以 `*/` 结尾
- **代码行**: 既不是空行也不是注释行

**实现细节**:
- 使用 `readFileSync` 读取文件内容
- 按 `\n` 分割成行
- 逐行判断并计数

### 4. 目录聚合器

**功能**: 按目录分组统计数据

**实现细节**:
- 从文件路径中提取目录名
- 使用对象或 Map 按目录分组
- 对每个目录聚合统计数据
- 计算总体统计

### 5. 输出格式化器

**功能**: 生成美观的控制台输出

**输出结构**:
```
======================================================================
📊 src 目录代码统计
======================================================================

📈 总体统计

   总文件数: 55
   总行数: 7,686
   有效代码行: 6,105
   空行: 1,164
   注释行: 417

📂 按目录统计

   agent           文件:   4  代码:    332  总计:    410
   cards           文件:   1  代码:    281  总计:    332
   ...

======================================================================
✅ 有效代码行数: 6,105
======================================================================
```
